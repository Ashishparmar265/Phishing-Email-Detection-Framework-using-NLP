# Codebase Explanation

This document provides a detailed breakdown of every module in the Phishing Email Detection Framework.

---

## 1. `src/preprocessing/data_pipeline.py`
- **Purpose**: Cleans raw, "noisy" email data for the model.
- **Key Class**: `DataPreprocessor`
    - `strip_html()`: Uses `lxml` parser to extract text from HTML bodies.
    - `clean_text()`: Standardizes the string format.
    - `tokenize_and_lemmatize()`: Uses NLTK's `WordNetLemmatizer` to reduce the feature space by grouping word variations.

## 2. `src/extraction/feature_extractor.py`
- **Purpose**: Extracts manual features that are strong indicators of phishing.
- **Key Class**: `FeatureExtractor`
    - `url_pattern`: A robust regex to find all hyperlinks.
    - `mask_sensitive_info()`: The "secret sauce" that replaces real URLs with `<URL>` tokens, forcing the model to learn the *context* of a link rather than the link itself.
    - `get_urgency_score()`: A custom metric that measures how "pressuring" the email sounds.

## 3. `src/classification/train.py`
- **Purpose**: The script that orchestrates model training.
- **Logic**:
    1. Loads the synthetic data generated by `data_loader.py`.
    2. Runs the extraction and preprocessing pipeline for every sample.
    3. Uses `TfidfVectorizer` with `max_features=5000` to limit complexity.
    4. Combines text vectors with custom features using `np.hstack`.
    5. Trains a `RandomForestClassifier` and saves it using `joblib`.

## 4. `src/classification/lstm_model.py`
- **Purpose**: Defines a deep learning alternative to the baseline.
- **Architecture**:
    - **Embedding Layer**: Converts words into dense vectors of fixed size.
    - **Bi-Directional LSTM**: Reads sequences in both directions to understand the "flow" of a phishing attempt (e.g., "If you don't [A], then [B] will happen").
    - **Dropout**: Prevents the network from becoming too specialized to the training data.

## 5. `src/api/main.py`
- **Purpose**: Serves the model as a RESTful web service.
- **Technology**: `FastAPI`
    - `@app.post("/predict")`: Accepts a JSON body and returns the full analysis.
    - **Lifecycle**: It loads the `rf_model.pkl` and `vectorizer.pkl` once at startup to ensure low-latency predictions.

## 6. `src/utils/data_loader.py`
- **Purpose**: Utility for development.
- **Logic**: Generates a balanced set of phishing and ham templates with randomized IDs and IPs to ensure the pipeline can be tested without downloading GBs of real data.
